# 训练输出说明

训练入口为：

```bash
uv run src/train.py
```

终端使用 Rich 表格显示训练诊断和实际生成效果。颜色用于快速定位趋势和异常，不改变任何指标数值。

## 先看哪一项

优先看“完整验证集实际生成效果”中的“综合 F1”。它是 Tap/Touch 起点 F1 与持续音起点 F1 的平均值，也是 `best_gen.pt`、提前停止和最终测试所使用的指标。

训练损失和验证损失用于判断优化是否收敛，但不能直接代表生成谱是否可用。

## 训练诊断表

| 列 | 含义 |
|---|---|
| 训练损失 | 使用随机数据增强的训练集损失。 |
| 验证损失 | 不更新参数时的验证集损失，最低值保存为 `best.pt`。 |
| 数量损失 | Tap/Touch 数量及持续音起点数量的分类损失。 |
| 时长损失 | 真实持续音起点上的时长回归损失。 |
| 学习率 | 下一轮使用的学习率。 |
| 生成未改善 | 完整验证集综合 F1 连续没有刷新最佳值的轮数。达到配置的耐心轮数时提前停止。 |

不再显示“完整时间帧命中率”。200 Hz 时间轴中绝大多数帧没有音符，该指标会被空帧严重抬高，容易误判模型效果。

## 实际生成效果

验证输出先经过与正式推理相同的 `events_to_frames()` 后处理，再与真实简化事件时间轴比较。

| 指标 | 含义 |
|---|---|
| 综合 F1 | Tap/Touch 起点 F1 和持续音起点 F1 的平均值。 |
| Tap F1 | Tap 与 Touch 合并后的起点匹配 F1。 |
| 持续起点 F1 | Hold、Touch Hold 和 Slide 合并后的起点匹配 F1。 |
| 严格 F1 | 必须落在同一个 5 ms 帧的综合 F1，用于观察精确时序。 |
| 时长 MAE | 只在成功匹配的持续音起点上计算平均绝对时长误差。 |
| 数量偏差 | 全部生成音符数相对真实音符数的偏差，正数表示过密，负数表示过稀。 |
| 过滤率 | 因短音间隔过小而被正式后处理丢弃的 Tap/Touch 比例。 |
| 空谱率 | 没有生成任何音符的验证谱面比例。 |

主 F1 允许前后 10 ms 的一对一匹配。重复预测不能重复命中同一个真实音符。严格 F1 仍保留，用于防止宽松匹配掩盖时序漂移。

持续音漏检只反映在起点召回率中，不会再次作为零时长写入时长 MAE。

## 精确定数表

浮点难度使用 `src/music_data.py` 从 Diving-Fish 歌曲数据取得的精确 `ds`，不是 `maidata.txt` 中将 `13+` 近似为 `13.5` 的粗略等级。

每个统计单元格格式为：

```text
生成平均/生成中位数 | 基准平均/基准中位数 平均偏差
```

基准来自本次训练扫描后全部有效谱面中相同精确 `ds` 的统计。表格中的“验证”是本轮验证谱面数，“基准”是完整基准样本数。

颜色规则：

- 绿色：生成平均值与基准平均值偏差不超过 10%。
- 黄色：偏差在 10% 到 25% 之间，或当前定数只有少于 3 张验证谱面。
- 红色：偏差超过 25%。

颜色只评价生成密度是否接近同定数总体分布，不代表音符时间匹配正确。实际正确性仍以 F1 为准。

## 指标边界

当前模型输出的是简化事件时间轴，因此这些指标不能评价：

- 具体键位是否正确。
- Touch 的具体区域是否正确。
- Slide 的形状、方向和终点是否正确。
- Break、EX 等属性是否正确。
- 谱面是否具有良好手感和可玩性。

当前 F1 只能说明音符起点时间、简化类别和持续时间的接近程度，不能视为完整 maimai 谱面还原分数。

## 检查点

| 文件 | 选择规则 |
|---|---|
| `checkpoints/newest.pt` | 每轮训练结束后的最新状态。 |
| `checkpoints/best.pt` | 验证损失最低。用于优化诊断。 |
| `checkpoints/best_gen.pt` | 完整验证集综合 F1 最高。正式推理和最终测试使用它。 |

评估口径带有版本标记。旧 checkpoint 缺少当前评估状态时不能用于恢复训练，但仍可按原架构用于单独推理。

## OVERFIT 模式

当 `config.yaml` 中的 `过拟合歌曲数` 大于 0 时，训练集和验证集是同一批歌曲。终端会显示红色警告。

这种模式只能用于确认模型能否记住少量歌曲，不能说明对未见歌曲的泛化效果。要进行正常训练和测试，应将该配置改为 `0`。
